# Alibi Foundation Document

## Core Premise

Alibi is not a voice memo app.

A voice memo app stores recordings. Alibi ingests raw thought, digests it, extracts signal, develops direction, and turns it into structured creative output.

The product begins with what the user says, writes, or uploads. Then it helps them decide what matters, what belongs together, what it could become, and how far they want to push it.

**Core promise:** Turn raw thought into organized, voice-aware, creatively expandable work.

---

## Pillar 1: Capture

This is the intake layer. The basis of all creation.

### Inputs

- Voice memos
- Typed text
- Long-form video
- Short video clips
- Uploaded transcripts
- Mixed media notes

### Key functions

- One-tap recording
- Typed entry
- Video upload
- Draft save
- Untitled quick-save flow
- Optional tags
- Optional project assignment
- Private by default
- Timestamped entries

### Principle

The user should not have to fully know what they’re making before they record.

---

## Pillar 2: Digestion

This is the first intelligence layer.

### Purpose

Convert raw material into workable signal without forcing the user to decode everything manually.

### Key functions

- Audio and video transcription
- Text cleanup
- Segmentation of long recordings
- Theme detection
- Emotional tone detection
- Repeated-idea detection
- Sensitivity and privacy flagging
- Story fragment detection
- Highlight-worthy line detection
- Incremental analysis as more material is added

### Principle

Digestion should reduce chaos without flattening the user’s signal.

---

## Pillar 3: Extraction

This is where the app pulls usable structure from raw input.

### What extraction should pull

- Core ideas
- Supporting points
- Stories and anecdotes
- Themes
- Emotional undertones
- Repeated language
- Quotes and hooks
- Character seeds
- Chapter or episode material
- Entertainment angles
- Educational angles

### Key functions

- Theme extraction
- Hook extraction
- Quote extraction
- Premise extraction
- Story seed extraction
- Contradiction and overlap detection
- Ranking of strongest ideas
- Ranking of most original ideas
- Suggestions for likely project fit

### Principle

Extraction should answer: what is actually worth building on?

---

## Pillar 4: Organization

This is the control layer.

### Structure

- Workspace
- Project
- Collection / Folder
- Entry
- Extracted ideas
- Selected assets
- Output drafts

### Key functions

- Drag and drop organization
- AI-suggested placement
- Manual override
- Smart collections
- Project-specific context bundles
- Selected / excluded states
- Canon or approved-source state
- Search across recordings and extracts
- Related-entry suggestions

### Principle

The app must prevent the user’s library from becoming a junk drawer.

---

## Pillar 5: Conceptualization

This is where raw signal turns into direction.

### Purpose

Help the user see what their material could become.

### Key functions

- Format suggestions
- Premise generation
- Theme boards
- Hook generation
- Narrative angle suggestions
- Educational angle suggestions
- Entertainment angle suggestions
- Tone and style direction cards

### Required creative play

This layer must go beyond summary. It should surface things users actually want to try.

Examples:

- This could become a mystical fiction concept
- This could become a morning-show style segment
- This could become a comedic host reaction
- This could become a character-driven dialogue
- This could become a polished entertainment format instead of a reflective memo

### Principle

Conceptualization should increase desire, not just understanding.

---

## Pillar 6: Creative Direction

This is the expansion layer.

### Purpose

Guide transformation into something more intentional, stylized, or surprising.

### Key functions

- Tone controls
- Format controls
- Style boards
- Guided ideation
- "Make this funnier"
- "Make this more dramatic"
- "Make this more nonfiction"
- "Make this more commercial"
- "Turn this into a show segment"
- "Turn this into chapter material"
- "Build a better concept from this"

### Trend-aware transformation templates

Alibi should include highly attractive transformation templates that make users want to test what their recordings can become.

Examples include:

- Late-night monologue energy
- Morning-show or panel-style commentary
- Advice-show format
- Sports desk or reaction-show breakdown
- Reality-show recap structure
- Fiction and mystical book directions
- Character-driven story generation
- Satirical and comedic reframing
- Trend-aware entertainment formats

These should be presented as recognizable creative vibes and structures, not literal copies of specific real shows or personalities.

### Principle

Creative direction should feel smart, useful, and alive, not random.

---

## Pillar 7: Concept Studio

This is the development room.

### Purpose

Give the user a polished place to actively build ideas, not just receive outputs.

### What happens here

The user selects a set of recordings, extracts, or themes and opens a creative session.

The app helps them:

- Find the premise
- Build the concept
- Choose the tone
- Compare directions
- Create a project brief
- Outline chapters or segments
- Build characters or frameworks
- Refine ideas in back-and-forth

### Key functions

- Idea cards
- Direction comparison
- Saved concept routes
- Tone board
- Premise builder
- Outline builder
- Character and story world builder
- Project brief generator
- Working session chat
- "Generate 3 stronger directions"
- "Push this further"

### Principle

This should feel like a premium development room, not a generic chatbot window.

---

## Pillar 8: Multi-Voice and Character Intelligence

This is the continuity layer for richer creative outputs.

### Purpose

Support multiple voices, characters, and conversational structures over time.

### Key functions

- Distinguish different speakers when needed
- Support multiple character voices inside stories
- Support multi-person podcast or dialogue concepts
- Define each speaker or character’s role, tone, and perspective
- Preserve continuity across repeated appearances of the same character or speaker
- Support structures where multiple personalities interact

### Story intelligence requirements

For fiction, the system should eventually understand:

- Recurring characters
- Character traits
- Dialogue tendencies
- Relationships and dynamics

### Show intelligence requirements

For podcast or show-style outputs, the system should eventually understand:

- Co-host formats
- Opposing viewpoints
- Recurring segments
- Speaker-specific delivery styles

---

This is where work gets made.

### Output types

- Long-form article
- Essay
- Memoir passage
- Chapter draft
- Nonfiction book outline
- Fiction story seed
- Podcast episode outline
- Podcast monologue
- Show segment
- Comedy bit
- Morning-show style breakdown
- Educational framework
- Social thread
- Script
- Spoken narration draft

### Key functions

- One source, many output types
- One output, many tone variants
- Versioned drafts
- Regenerate with changed direction
- Source-linked output
- Included-material visibility
- "Stay close to source" mode
- "Expand creatively" mode
- "Invent further" mode
- Private draft vs public-ready mode

The output engine should produce meaningfully different, useful outcomes, not shallow rewrites.

---

## Pillar 10: Packaging and Export

Packaging is not secondary. It is part of what makes the product desirable.

### Purpose

Turn internal creative work into something usable, portable, and presentable.

### Key functions

- Copy export
- Structured text export
- Script export
- Chapter export
- Episode export
- Title generation
- Description generation
- Cover concept prompting
- Public post packaging
- Product packaging later

### Packaging requirement

The app should make transformation feel like entering a real format or creative lane, not just pressing a generic generate button.

The experience should create the feeling that a raw voice note can become something cinematic, entertaining, intelligent, stylized, or publishable.

---

## Security, Privacy, and Voice Protection

Because the platform may eventually involve highly personal recordings, identity-linked content, and voice-based creative outputs, a secure foundation is critical.

### Core requirements

- Raw recordings are private by default
- Users control what gets included in projects, outputs, or public work
- Sensitive material is never assumed safe for output
- Voice-related features require strong permission boundaries
- Multi-person systems require clear access and usage rules
- The app clearly distinguishes raw source material, approved source material, inferred material, and invented output

### Principle

Trust is part of the product.

## Recommended Lifecycle

1. Capture
2. Digest
3. Extract
4. Organize
5. Conceptualize
6. Direct
7. Develop
8. Generate
9. Package

---

## The Three Main Product Modes

### Mode A: Capture + Understand

"I want to get thoughts out and have the app make sense of them."

### Mode B: Transform + Create

"I know roughly what I want. Help me turn this into something."

### Mode C: Develop + Invent

"I have fragments. Help me discover the stronger idea and build it."

---

## Strategic Product Truth

Alibi wins if it becomes good at this exact sequence:

**Capture → Digest → Extract → Organize → Conceptualize → Direct → Develop → Generate → Package**

A voice memo app stores what you said. Alibi figures out what it means, what matters, and what it could become.
