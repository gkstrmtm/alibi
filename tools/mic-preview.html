What this solves

This gives a clean mic selector for testing. The native Expo app on Android emulator does not give a custom in-app Windows microphone picker.

1. Use this page to confirm which mic actually works.
2. In Windows Sound Settings, set that same microphone as the default input device.
3. Then test recording in the emulator. The emulator generally follows the host/default input path.
If device labels are blank at first, that is normal until permission is granted.
If the meter still does not move after selecting a device, the issue is outside the app layer.
